Why yajc.js

Because other carousels, although several being very excellent, don't do exactly what I need and customizing them is so painful. After a couple of days trying to integrate 5 or 6 different carousels into a project I worked on without fruitful result, I gave up and coded yajc.js in a few hours. Who says NIH is never a good thing?

Feature highlights

  • Single-minded: a minimally programmable horizontal carousel, that's it
  • Responsive to however-styled containing HTML element and browser resize
  • Small, just over 2KB of gzipped JavaScript and CSS
  • Tested on Chrome, Firefox, Safari 5/6/Mobile, Internet Explorer 9/10

yajc.js currently does not support guestures like flick, wipe etc. on touch devices. Also, some examples below don't work when hovering or double-clicking are used.

HTML
<div id='c1'></div>
JavaScript
$(function() {
    $carousel = $('#c1').yajc() 
        .yajc('wings') 
        .yajc('clickSelect');

    $.getJSON('images/images.json', function(images) {
        (function load(index) {
            var img = images[index % images.length];
            $carousel.yajc('add', '<li><img src="' + img.url + '"></li>', function() {
                if (++index < 30) load(index);
            })
        })(0);
    });
});

Core

yajc.js has a very small core, exposing 6 methods:

  • ctor ([options]): invoke by calling $obj.yajc({...}). Options include:
    • addDuration: the duration in ms to animate element addition
    • removeDuration: the duration in ms to animate element removal
    • msPer100px: the duration in ms to animate every 100 pixels, used to control sliding speed
    • showEasing: the name of the easing function used in show()
    • slideEasing: the name of the easing function used in slide()
    • You can set global defaults by overriding properties in $.fn.yajc.defaults
  • add (target [, position] [, callback])
    • target: the jQuery object or DOM element to be added
    • index (optional): the index to add to, default is the end of the list
    • callback (optional): the callback to be invoked after adding
  • remove (target [, callback])
    • target: the jQuery object, DOM element or index to be removed
    • callback (optional): the callback to be invoked after removing
  • show (target [, callback])
    • target: the jQuery object, DOM element or index to be shown
    • callback (optional): the callback to be invoked after showing
  • slide (to [, callback])
    • to: the position or offset (if prefixed with += or -=) to slide to
    • callback (optional): the callback to be invoked after sliding
  • refresh ([callback]): refresh the carousel if there is any external change, e.g. add elements without using the add method.

You can write JavaScript code invoking these methods to control the behaviors of the carousel. You can also do it by coding plugins.

Plugins

You can code yajc.js plugins by adding methods to $.fn.yajc.plugins. yajc.js comes with 3 built-in plugins: wings, clickSelect and hoverSlide. You have seen wings and clickSelect in the first example. Example using hoverSlide is as follows.

HTML
<div id='c2'></div>
JavaScript
$(function() {
    $carousel = $('#c2').yajc().yajc('hoverSlide');
    loadImage($carousel);
});

More examples

Use options

HTML
<div id='c3'></div>
JavaScript
$(function() {
    $carousel = $('#c3').yajc({
        addDuration: 100,
        msPer100px: 300
    }).yajc('clickSelect');
    loadImage($carousel);
});

Delete by double-clicking

HTML
<div id='c4'></div>
JavaScript
$(function() {
    $carousel = $('#c4').yajc({
        removeDuration: 300
    }).yajc('wings', 'circle');

    $carousel.find('li').live('dblclick', function() {
        $carousel.yajc('remove', this);
    });
    
    loadImage($carousel);
});
